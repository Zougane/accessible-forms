<input> with visual "required" indicator

← Return to main list

This is a test case designed to determine how a visual "required" indicator is announced by various screen readers.

Example

Code

<div>
  <label for="name">Full name (Required)</label>
  <input required type="text" id="name">
</div>

Assistive technologies

VoiceOver

  • Chrome: Full name. Required. Required. Invalid data. Edit text.
  • Firefox: Full name. Required. Required. Invalid data. Edit text.
  • Safari: Full name. Required. Required. Edit text.

NVDA

  • Chrome: Full name. Required. Edit. Required. Invalid entry. Blank.
  • Firefox: Full name. Required. Edit. Required. Invalid entry. Has autocomplete. Blank.
  • Edge: Full name. Required. Edit. Required. Invalid entry. Blank.

JAWS

  • Chrome: Full name. Left parent. Required. Right parent. Edit. Required. Invalid entry. Type in text.
  • Firefox: Full name. Left parent. Required. Right parent. Edit. Required. Invalid entry. Type in text.
  • Edge: Full name. Left parent. Required. Right parent. Edit. Required. Invalid entry. Type in text.
